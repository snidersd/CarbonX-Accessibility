Accessibility Considerations

  • Tabs should activate automatically when they receive focus as long as their associated tab panels are displayed without noticeable latency.
  • Tabs that are arranged vertically should use the Up and Down Arrow keys to navigate the tab list.

Helpful Resources for Creating Customized Accessible Implementations:

  • W3C WAI-ARIA Tab Design Pattern covers the usage of ARIA names, state and roles, as well as the expected keyboard intractions.
  • IBM Accessibility Checklist - Checkpoint 1.3.1 Info and Relationships includes the development techniques to meet this checkpoint.
